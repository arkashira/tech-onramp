# Business Model Canvas – tech‑onramp

| **Section** | **Details** |
|-------------|-------------|
| **Value Proposition** | • **Modular, Markdown‑driven curriculum** that lets educators author learning paths without tooling overhead.<br>• **Zero‑configuration deployment** – load modules from a folder, parse front‑matter, and publish instantly.<br>• **Rapid iteration** – changes to Markdown files are reflected on the next run, enabling continuous content improvement.<br>• **Open‑source & extensible** – plug‑in new renderers, analytics, or delivery channels with minimal friction. |
| **Customer Segments** | 1. **EdTech startups** building micro‑learning platforms.<br>2. **Corporate training teams** needing quick, versioned content updates.<br>3. **University departments** that want a lightweight LMS for niche courses.<br>4. **Independent educators & bootcamps** looking for a free, self‑hosted curriculum engine. |
| **Channels** | • **GitHub repository** (source & documentation).<br>• **PyPI package** (`pip install tech-onramp`).<br>• **Community forums** (Stack Overflow, Reddit r/edtech).<br>• **Tech blogs & newsletters** (e.g., Medium, Hacker News).<br>• **Conferences & meetups** (PyCon, EdTech Expo). |
| **Revenue Streams** | 1. **Premium support & consulting** for enterprise deployments.<br>2. **Hosted SaaS offering** (tech‑onramp‑cloud) with managed hosting, analytics, and integrations.<br>3. **Marketplace add‑ons** (custom renderers, analytics plugins) sold via the PyPI ecosystem.<br>4. **Training & certification** courses for educators on using the engine. |
| **Cost Structure** | • **Development & maintenance** (core team, CI/CD, documentation).<br>• **Cloud infrastructure** for SaaS (compute, storage, CDN).<br>• **Marketing & community engagement** (content, events).<br>• **Support & SLAs** (ticketing, uptime monitoring). |
| **Key Resources** | • **Open‑source codebase** (Python, pytest, Markdown parsing).<br>• **Documentation & examples** (GitHub README, tutorials).<br>• **Community contributors** (issue triage, PR reviews).<br>• **CI/CD pipelines** (GitHub Actions). |
| **Key Activities** | • **Feature development** (new module types, rendering engines).<br>• **Quality assurance** (unit & integration tests).<br>• **Community engagement** (answering questions, gathering feedback).<br>• **Product marketing** (blog posts, webinars). |
| **Key Partners** | • **Python ecosystem** (PyPI, setuptools).<br>• **Markdown tooling** (markdown‑python, front‑matter libraries).<br>• **Hosting providers** (AWS, GCP, Azure) for SaaS. <br>• **Learning‑management systems** (Moodle, Canvas) for integration partners. |
